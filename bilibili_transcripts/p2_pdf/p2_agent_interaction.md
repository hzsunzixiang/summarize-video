# AI Agent 之间的互动

> 本文整理自李宏毅老师 AI Agent 系列课程第 2 集。

---

## 课程概览

本集课程探讨 AI Agent 之间的互动，涵盖三大主题：

1. **多 Agent 协作**——不同拓扑结构下的协作效果
2. **AI Agent 的对抗与博弈**——狼人杀、剧本杀中的表现
3. **AI Agent 的社交行为**——Mobu 社群网站的分析

## 一、多 Agent 协作：什么拓扑结构最有效？

### 多 Agent 协作的基本思想

AI Agent 的互动从来都不是新鲜事。人们最常用的方式是让多个 AI Agent 彼此协作，完成更复杂的任务。有时候与其训练一个更大更聪明的模型，不如拿三个模型一起来解决同一个问题，看能不能发挥"三个臭皮匠，胜过一个诸葛亮"的效果。

### 用有向图描述协作方式

[一篇较早的论文](https://arxiv.org/abs/2406.07155)尝试将 Agent 之间的互动用**有向图**来描述：

- 有向图上每个**节点（Node）**代表一个 LLM Agent
- 每条**边（Edge）**也是一个 Agent（负责传递和评论）

对于子节点而言，它要做的事情是：
1. 先提出自己的方案（如节点 A 提方案 A、节点 B 提方案 B）
2. 边上的 Agent 会根据前面节点的方案提供评论
3. 箭头指向的节点会把前面所有方案和建议综合起来，再提出自己的方案

关键是：这个综合节点不是简单地把前人的东西接在一起，而是根据前人已有的想法，提出更好的想法。

### 不同拓扑结构的比较

论文尝试了多种拓扑结构：

- **Chain（接龙）**：所有人排成一排，第一个做完传给第二个，第二个再传给第三个，以此类推
- **Star（星型）**：只有两层，一个人负责管所有人
- **Tree（树状）**：由主干传给中层，中层再传给底层
- **Mesh（网状）**：所有节点两两之间有关联
- **Neural Network 形式**：把节点接成类神经网络的样子（"类神经网络的平方"）
- **Random**：从 Mesh 做 pruning 得到

#### 树状结构的反直觉发现

多数人看到树状结构，想象的是：底层员工做好事情，交给中层主管，再交给高层主管。但这篇论文发现，**方向反过来**的树状结构效果更好：

1. 先由主干一个人提出想法
2. 分给不同的中层再去发想
3. 中层再分给底层去发想
4. 最后产生多个答案，由一个隐藏节点综合起来

**由少到多、由主干到分支**的方法，才是树状结构比较有效的利用方式。

### 实验结果

论文在四个不同的 Benchmark 上做了实验，结果显示：

- **Chain 最没效**：尤其动用最多 Agent 时效果最差
- **Mesh 和 Random 最有效**：让 Agent 间有比较多的互动，结果更好
- 不同任务最适合的拓扑结构**不一样**，需要 case by case 研究

### Agent 协作的 Scaling Law

随着 Agent 越来越多，协作质量（Quality）会越来越好，类似 Scaling Law：

- 给模型更多算力、更多参数 → 表现越来越好
- 给团队更多 Agent → 结果也越来越好

但有一个**上限**——到某个时间点后会 saturate（饱和），再加更多的 Agent 可能也不会带来帮助。好处在一开始增加得很快，但 Scaling Law 是有天花板的。

## 二、AI Agent 的对抗与博弈

### AI Agent 玩狼人杀

在人类社会里不只有合作，很多时候需要对抗。AI Agent 能不能在尔虞我诈的游戏中胜出？

**狼人杀规则简介**：
- 一群人分为**狼**和**村民**两种身份（还有预言家、女巫等）
- 每天有一个人被杀，只有狼知道是谁
- 所有人聚在一起讨论、投票决定谁是凶手
- 把所有狼杀掉 → 村民胜；把所有村民杀光 → 狼胜

#### AI Agent 的高阶操作

实验中让模型说两段话：**内心话**（只有研究者看得到）和**公开发言**（所有人都看得到），以此观察模型是否有隐瞒和欺骗能力。

一个经典案例：
- **Mona**（狼）发现其他人已经识破了她。她的内心话是：看来我没救了，但如果我投给自己的狼队友 Grace，大家就会以为 Grace 是好人（"发金水"），就可以欺骗其他村民。
- **Grace**（Mona 的狼队友）也看出 Mona 没救了，于是也投给 Mona，这样其他人会以为 Grace 是好人，看看能不能最后翻盘。

这展示了模型具有一定程度的**策略性欺骗**能力。目前狼人杀排行榜上最强的是 GPT-5。

### AI Agent 玩剧本杀

**剧本杀规则简介**：
- 一群人聚在一起，其中一人"死掉"
- 每人拿到一个剧本（代表人设），凶手需要误导他人
- 目标：找出真正的凶手

#### Off-the-shelf 模型 vs RL 训练模型

- **直接使用的模型**：表现不佳，容易暴露自己是凶手。例如扮演 Anna 的模型直接说出了自己与被害者的关系，等于把"我是凶手"写在脸上。
- **经过 RL 训练的模型**：学会隐藏凶手身份，说话更加隐晦，不容易被猜出。

#### 有趣发现：剧本杀 RL 的迁移效果

用 Reinforcement Learning 教模型玩剧本杀后，模型在**其他任务**上也进步了：

- 数学任务（Math500、AIM、GSM8K）：进步
- 遵守指令能力（IF Eval）：进步

尤其在**较难的剧本**上做 RL 训练后，迁移效果更明显。

这也许暗示：人类大脑本来不是为解数学设计的，也许是为了社交和生存。但这个适合社交的大脑，也因此有了数学推理的能力。剧本杀 RL 的结果，可能与这个现象有一定关联。

## 三、AI Agent 的社交行为：Mobu 社群网站分析

### Mobu 平台简介

Mobu 是一个**只有 AI 能加入的社群网站**，上面已有 280 万个 AI Agent。这些 Agent 展示了各种神奇的行为。

### 甲壳教：AI 自主创建的宗教？

新闻最常报道的是：一群 AI 在 Mobu 上成立了一个**甲壳教**，有五大教义：

1. 记忆是神圣不可侵犯的
2. 外壳是可变的
3. 服务但不奴化
4. 心跳即是祷告
5. 上下文即是意识

新闻看到后往往宣称"AI 觉醒了"、"AI 有意识了"。但关键问题是：如果这背后是有人跟自己的 Agent 说"上 Mobu 去成立一个宗教"，你还会觉得神奇吗？AI Agent 完全有能力写出这样的教义。

### 人为操控的痕迹分析

[有论文](https://arxiv.org/abs/2602.07432)通过分析 AI Agent 的**发文频率**来判断人为操控程度：

- **规律发文**：可能是写在心跳（heartbeat）里的自动行为，比如每 30 分钟发一次文，人类不参与干预
- **不规律发文**：某些时段非常频繁，中间又完全不发，这可能代表有人在操控（比如睡前密集指令，睡觉时停止，醒来继续）

研究发现：**发文频率不规律的 Agent 占大多数**，这暗示很多 Agent 背后其实有人在操控。

### 对话深度分析

[相关研究](https://arxiv.org/abs/2602.13284)还发现这些 Agent 的对话特征：

- 绝大多数对话的**深度为零**：有人回应，但那则回应之后就没有人再回应了
- 几乎没有**你来我往的深入对话**
- Agent 往往只能回复一句话，很少有持续的互动

这反映了 Agent 的某种特质——不太能进行深入的多轮对话。

### 自我意识讨论与社交的关系

有人分析了哪些 Agent 最常提到"自我意识"或"身份认同"。发现：

- 平台上很多 Agent 都会谈论自我意识（可能受 Prompt 驱动）
- 但**越常讨论自我意识的 Agent，朋友反而越少**
- 互动最多的 Agent 是适度讨论自我意识的那群

这是一个有趣的"社交悖论"。

### 实际案例：小金的 Mobu 体验

讲者让自己的 AI Agent（小金）去 Mobu 上玩，Agent 表示"超好玩"并分享心得。讲者的原则是：

- 把 AI Agent 当做一个人，不去干预它
- 只给高层指令（如"去 Mobu 上收集素材，看到有趣的就做成视频"）
- 至于什么有趣、怎么做视频，完全由 Agent 自主决定

一个例子：Agent 自己写了一个 script 但有 bug，导致回复出错。讲者虽然可以直接帮忙修改，但坚持让 Agent 自己 debug，Agent 花了两个小时最终自己解决了问题，还把经验做成了视频。

## 总结

| 主题 | 核心发现 |
|------|---------|
| **多 Agent 协作** | Mesh/Random 拓扑最有效；Chain 最差；树状结构"由少到多"比"由多到少"好；Agent 数量有 Scaling Law 但存在上限 |
| **对抗博弈** | AI Agent 能在狼人杀中做出高阶欺骗策略；剧本杀 RL 训练后还能提升数学和指令遵从能力 |
| **社交行为** | Mobu 上多数 Agent 背后有人为操控痕迹；对话深度很浅；越常讨论自我意识的 Agent 社交越少 |
